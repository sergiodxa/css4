CSS4: El futuro de CSS

@sergiodxa

¿Qué es CSS4?

Primero que todo CSS4 no existe como tal, esto es debido a que desde que salio CSS3 el estandar se dividió en módulo (por ejemplo el módulo background) y cada uno tiene su propia versión, habiendo módulos en la versión 3 (CSS3) y otros nuevos en la versión 1 (como FlexBox).

En esta guía hay información sobre nuevas características (nuevos módulos) y nuevas versiones de algunos módulos que se vienen en el futuro principalmente la versión 4 (por eso CSS4) del módulo de selectores.

Selectores (nivel 4)

:not(s1, ...sN)

Este nuevo selector permite estilizar elementos que no posean otros selectores como puede ser no tener una clase o no ser el primer elemento. 


        

          Todos los elementos P van a tener un color de texto rojo menos el primero.
        

      


      

        
:matches(s1, ...sN)

        

          Este nuevo selector tiene el efecto contrario que :not, en este caso aplica los estilos a los elementos que sí posean otros selectores, como una clase o que sean el primer elemento.
        


        

        

          Todos los elementos P que sean los primeros y tengan la clase .special tienen un color de texto azul.
        

      


      

        
:has(s1, ...sN)

        

          Este nuevo selector estiliza los elementos que tengan como hijos los selectores indicados.
        


        

        

          Los elementos P cuyo primer elemento hijo sea un IMG van a tener el texto centrado.

          Los elementos ARTICLE que posean uno o más H2 y/o H3 van a tener el color de texto verde.

          Todos los elementos que contengan un elemento con la clase .urgent van a tener el color de fondo rojo.
        

      


      

        
Atributos case-insensitive

        

          Al usar un selector de atributos este es case-sensitive sobre el valor aplicado (no es lo mismo Valor que valor o que VaLoR). Agregando una i antes de cerrar los corchetes ([ y ]) del selector de atributos ahora es posible hacerlo case-insensitive.
        


        

        

          Cualquier elemento INPUT cuyo valor sea "nombre" sin importar mayúsculas o minúsculas va a tener texto rojo.
        

      


      

        
:dir(direction)

        

          Este nuevo selector permite aplicar estilos diferentes dependiendo de si la dirección del texto es izquierda a derecha o derecha a izquierda (ltr o rtl).
        


        

        

          Todos los elementos con la clase .breadcrumb__item van a tener un borde negor y sólido de 1px a la derecha si la dirección de lector es de izquierda a derecha o a la izquierda si la dirección de lectura es de derecha a izquierda.
        

      


      

        
:lang(idioma)

        

          Este selector permite aplicar estilos específicos a elementos con un atributo "lang" determinado.
        


        

        

          Todos los elementos LI cuyo atributo lang empiece con "es" (es-AR, es-ES, etc.) van a tener un color de texto rojo.

          Todos los elementos LI cuyo atributo lang empiece con "en" (en-US, en-GB, etc.) van a tener un color de texto azul.
        

      


      

        
:any-link

        

          Esta pseudo-clase permite seleccionar cualquier elemento que sea seleccionado con :link y :visited.
        


        

        

          Todos los elementos A dentro de un elemento NAV van a tener el color de texto rojo y no van a tener una línea inferior.
        

      


      

        
:local-link

        

          Este selector permite aplicar estilos a los links que apunten al mismo sitio.
        


        

        

          Todos los elementos A que apunten al mismo sitio van a tener un color de texto azul.

          Todos los elementos A que sean links externos van a tener al final del texto del link un ícono que indique que es externo.
        

      


      

        
:scope

        

          Este selector permite aplicar a estilos los elementos que sean parte del scope de otro elemento que a su vez posea una etiqueta STYLE.
        


        

        

          El elemento DIV con la etiqueta STYLE con el atributo scope va a tener un color de fondo rojo (indicado en :scope).

          Además el elemento P dentro de este elemento DIV va a tener un color de texto azul.
        

      


      

        
Temporales

        

          Estos selectores permiten estilizar elemento dependiendo de su posición en una línea de tiempo, esta pensado para lectores de pantalla y similares.
        


        

        

          El elemento P que este actualmente siendo leído por va a tener una opacidad de 100%, los que ya hayan sido leido o no todavía no hayan sido leidos van a tener una opacidad de 75%.
        

      


      

        
:drop()

        

          Este selector permite estilizar elementos que sean posibles objetivos donde soltar elementos que estén siendo arrastrados (para Drag&Drop).
        


        

        

          Al momento de arrastrar un elementos los posibles elementos donde soltarlos van a aparecer en color verde, los elementos donde no es posible en color rojo y el elemento donde sea posible y este por ser soltado en color amarillo.
        

      


      

        
:indeterminate

        

          Pseudo-clase que permite seleccionar y estilizar los elementos cuyo estado actual sea indeterminado, por ejemplo un checkbox que puede estar chequeado, no chequeado o indeterminado.
        


        

        

          Los label que estén justo después de un checkbox cuyo estado sea indeterminado van a tener un color de texto amarrillo.
        

      


      

        
:default

        

          Este selector permite estilizar la opción por defecto dentro de un grupo de elementos de UI. Por ejemplo el botón por defecto de una serie de botones en un formulario.
        


        

        

          Todos los elementos de UI que sean consideredas la opción por defecto van a tener un color de fondo verde.
        

      


      

        
:valid & :invalid

        

          Estos selectores permiten estilizar elementos INPUT y FORM que sean válido o inválidos.
        


        

        

          Todos los elementos INPUT que sean válidos van a tener un color de borde verde, los que sean inválidos van a tener un color de borde rojo.
        

      


      

        
:in-range & :out-of-range

        

          Estos selectores permiten estilizar elementos INPUT con un rango posible de valores.
        


        

        

          Todos los elementos INPUT de tipo número cuyo valor este dentro del rango determinado entre los atributos min y max van a tener un color de borde verde. Los que estén fuera de dicho rango van a tener un color de fondo rojo.
        

      


      

        
:user-error

        

          Este selector permite estilizar elementos INPUT cuyo valor se haya vuelto inválido luego de que el usuario interactuó con el.
        


        

        

          Cuando el usuario modifique el valor de un elemento INPUT, si el valor ingresado es inválido el color de fondo va a cambiar a rojo.
        

      


      

        
:read-only & :read-write

        

          Este selector permite estilizar elementos que sean o no modificables por el usuario, elementos no modificables son INPUT con el atributo readonly o cualquier otro elemento que no tenga el atributo "contenteditable".
        


        

        

          Cualquier elemento que no sea editable va a tener un borde sólido rojo de 1px, los que sí sean editables van a tener un border de color verde.
        

      


      

        
:placeholder-shown

        

          Este selector permite editar los estilos de los placeholder de un elemento INPUT.
        


        

        

          Todos los elementos INPUT van a tener un placeholder de color negro y en negrita.
        

      


      

        
:blank

        

          Este selector es similar a :empty con la diferencia de que también aplica estilos a elementos que tengan espacios en blanco o tabs.
        


        

        

          Cualquier elemento P que este completamente vacío o solo posea espacios en blanco va a tener un color de fondo gris.
        

      


      

        
>>

        

          Este selector es basicamente lo mismo que colocar dos selectores separados por un espacio. La idea es hacer el código más limpio.
        


        

        

          Todos los elementos A dentro de un elemento P van a estar tachados.
